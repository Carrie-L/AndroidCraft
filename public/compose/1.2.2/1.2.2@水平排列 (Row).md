
# 1.2.2 水平排列 (Row)

## 1. 核心理论：X 轴上的排队逻辑

### 1.1 什么是 Row？
`Row`（行）是 `Column` 的兄弟。它的逻辑和 `Column` 完全一样，只是方向转了 90 度。
*   **使命**：把它的子元素，沿着 **X 轴（水平方向）**，从左到右依次摆放。
*   **对应 XML**：`LinearLayout` (`orientation="horizontal"`)

### 1.2 默认行为：基线对齐 (Baseline Alignment)
这是一个非常专业且贴心的细节。当你在 `Row` 里放两个不同字号的文字时：
*   文字 A：大号字体（30sp）
*   文字 B：小号字体（12sp）

如果机械地按顶部对齐，小字会挂在大字的上面，看起来很别扭。
Compose 的 `Row` 默认具备**基线对齐**的智能。它会尝试让文字的底部基准线（就像写英语字母时的那条四线格底线）对齐，这样看起来不仅整齐，而且符合阅读习惯。

### 1.3 溢出问题
`Row` 只是一个排列容器，它**不具备滚动功能**。
如果你的子元素太多，超出了屏幕宽度：
*   **View 体系**：可能会被挤压或截断。
*   **Compose**：元素会直接画到屏幕外面去，甚至可能什么都看不见（如果父容器限制了绘制区域）。
*   **解决**：如果你需要水平滚动，需要给 Row 加上 `Modifier.horizontalScroll`（后续修饰符章节会讲），或者使用 `LazyRow`（列表章节会讲）。目前我们只关注静态排列。

## 2. 知识测验 (Quiz)

### 问题 1 [代码填空]
你想做一个“点赞按钮”，左边是一个心形图标 `Icon`，右边是点赞数量 `Text`。你应该用什么容器包裹它们？

```kotlin
____ {
    Icon(...)
    Text("1024")
}
```

A. `Column`
B. `Row`
C. `Box`

> **答案：B**
> **解析**：因为图标和文字是左右并排显示的，所以必须用 `Row`。

### 问题 2 [视觉想象]
如果在 `Row` 里放了三个 `Text`，分别是 "A", "B", "C"。屏幕宽度足够大。它们会怎么显示？

A. A 在最上面，B 在中间，C 在最下面。
B. A 在最左边，B 在 A 的右边，C 在 B 的右边。
C. A, B, C 全部重叠在左边。

> **答案：B**
> **解析**：`Row` 的布局算法就是水平从左到右放置。

## 3. 编码实战 (Code Lab)

**任务目标**：利用 Row 和 Column 的组合，构建一个经典的“用户标签”组件。

我们要做成这样的效果：
```
[图标]  用户名
        用户等级
```
*   左边是一个方块（模拟头像）。
*   右边分两行：上面是用户名，下面是等级。

### 步骤 1：分析结构
这需要**嵌套布局**的思维。
1.  最外层：左（图）右（文），所以是 **水平排列 (`Row`)**。
2.  右边的部分：上（名）下（级），所以是一个 **垂直排列 (`Column`)**。

### 步骤 2：编写代码
在 `MainActivity.kt` 中定义 `UserTag` 组件。

```kotlin
@Composable
fun UserTag() {
    // 1. 最外层用 Row
    Row {
        // 左边：头像 (暂时用 Text 模拟方块)
        Text(text = " [头像] ")
        
        // 右边：信息区 (用 Column 包裹)
        Column {
            Text(text = "用户：Android大牛")
            Text(text = "等级：Lv.99")
        }
    }
}
```

### 步骤 3：运行与验证
在 `setContent` 中调用 `UserTag()`。
仔细观察效果：
*   “[头像]” 在最左边。
*   紧挨着它的右边，是两行文字。
*   这两行文字是上下对齐的。

这就是布局的核心奥义：**组合 (Composition)**。
通过简单的 Row 和 Column 的无限嵌套，你可以构建出世界上 99% 的复杂界面（剩下的 1% 交给 Box 和自定义布局）。

### 进阶挑战（思考题）
如果我把最外层的 `Row` 改成 `Column`，会变成什么样？
*   **推测**：头像会跑到最上面，名字和等级会跑到头像的下面。变成了垂直布局。
你可以动手改一下试试，验证你的推测。
